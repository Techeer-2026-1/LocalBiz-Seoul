# 1. 서비스 개요

## 1.1 프로젝트 소개

LocalBiz Intelligence는 **서울 로컬 라이프 AI 챗봇**이다. 사용자가 자연어로 질문하면 서울의 장소, 행사, 맛집, 카페 등에 대해 AI가 실시간으로 검색·추천·분석하여 응답한다.

핵심 차별점:

- **하이브리드 검색**: 정형(PostgreSQL + PostGIS) + 비정형(OpenSearch 768d k-NN) 동시 처리
- **4-Tier 데이터 계층**: 구조적 → 리뷰 → LLM Rerank → 큐레이션 위키
- **LLM 기반 분석**: 리뷰 6개 지표 런타임 채점, 코스 추천, 비용 견적
- **실시간 스트리밍**: WebSocket 기반 블록 순서 응답
- **서울 특화**: 53만 장소 + 7천 행사 + 28만 생활인구 데이터

## 2.3 데이터 흐름 (v2 — 4-Tier 검색)

```
사용자 쿼리 ("카공하기 좋은 카페 성수동")
  ↓ WebSocket
FastAPI + LangGraph
  ↓
[1단계: Intent Router] Gemini → 의도 분류 + 카테고리 추론 + 쿼리 확장
  ↓
[2단계: Retrieve — 후보 확보]
  ├─ SQL: category='카페' + district LIKE '%성동%' + PostGIS    → Tier 1 (구조적)
  ├─ Vector k-NN: places_vector (page_content 의미 매칭)       → Tier 2 (경험적)
  └─ place_reviews: 리뷰 키워드 매칭 ("카공", "콘센트")         → Tier 3 (리뷰)
  ↓ 합산 + 중복 제거
[3단계: LLM Rerank] Gemini Flash → 후보 재정렬 (상식 기반 판단)
  ↓
[4단계: 응답 생성] text_stream + places[] + map_markers
```

### 4-Tier 데이터 계층

| Tier | 데이터 | 커버리지 | 쿼리 유형 |
|---|---|---|---|
| **1. 구조적** | CSV raw_data + Layer 2 카테고리 설명 | 535K (100%) | "강남 치킨집", "무료 주차장" |
| **2. 경험적** | Naver Blog 리뷰 요약 + 키워드 | ~7K (진행 중) | "분위기 좋은 카페" |
| **3. LLM Rerank** | 런타임 Gemini 판단 | 전체 (런타임) | "야경이 예쁜 곳" |
| **4. 큐레이션** | YouTube/블로그 전문 + wiki 엔티티 | ~1,000 (진행 중) | "콘센트+조용+넓은 카페 성수" |

## 5.1 설계 원칙

- "공통 기둥 + 유연한 주머니(JSONB)" 패턴
- PostgreSQL `place_id` = OpenSearch `_id` (application 레벨 연결)
- 대화 이력 이원화: LangGraph checkpoint + messages (append-only)
- PK 전략: OpenSearch 연동 = VARCHAR(36), 나머지 = BIGSERIAL
- 3NF + 의도적 비정규화 4건 (places.district, events.{district,place_name,address}, raw_data JSONB)

# 7. 비정형 ETL 파이프라인

## 7.1 OpenSearch 임베딩

### Pipeline A: places_vector (535K)

- **입력**: PG places 전체
- **page_content 생성**: source별 특화 템플릿 (20종) + Layer 2 카테고리 기본 설명 (60종)
- **임베딩**: Gemini gemini-embedding-001 batchEmbedContents (100건/call)
- **적재**: OpenSearch _bulk
- **소요**: ~4시간 (순차), async 전환 시 ~3.5분

### Pipeline B: place_reviews (~7K 진행 중)

- **입력**: Naver Blog Search API (장소별 3건 리뷰 수집)
- **분석**: Gemini Flash → 6 지표 + 키워드 + 요약
- **필터링**: relevance 필터 (상호명 매칭 + 지역 필터) + 점수 ≤1.0 skip
- **임베딩**: summary_text → Gemini 768d
- **적재**: OpenSearch place_reviews
- **개선 이력**: v1(76%) → v2(52%) → v4(86%) → v6(72%) — 정밀도 vs 재현율 최적화

### Pipeline C: events_vector (7.3K)

- **입력**: PG events 전체
- **텍스트**: title + summary
- **임베딩**: Gemini 768d → OpenSearch events_vector

## 7.2 YouTube 큐레이션 데이터 (Tier 4)

### 수집 파이프라인

```
검색어 85종 (13 intent 전체 커버)
  → YouTube 검색 (yt-dlp ytsearch20)
  → 중복 제거 (video_id + 채널별 3건 제한)
  → 인기 우선, 중복률 >50%면 50건 확대
  → description (yt-dlp) + transcript (youtube-transcript-api)
  → Gemini Flash 구조화 추출 (장소/코스/features/keywords)
  → data/raw/youtube/ (불변 원본 .md)
  → data/extracted/youtube/ (구조화 JSON)
  → data/manifest.json (라우팅 인덱스)
```

### 추출 품질 (PoC 검증)
