# LocalBiz Intelligence — PoC 보고서 (EVENT_SEARCH)
**작성일:** 2026-03-25
**대상 기능:** 행사/이벤트 검색

---

## 1. 개요

서울 행사 정보를 제공하는 PoC로, 초기 서울 열린데이터광장 API에서 **네이버 뉴스+블로그 검색 기반 LLM 추출** 방식으로 피벗했다.

**검증 완료 항목**
- 다양한 날짜 표현 파싱 (이번 주말, 다음 주, N월 초, 봄 등 15+ 패턴)
- 날짜 기반 동적 검색 쿼리 자동 생성 (3개 쿼리)
- 네이버 뉴스 + 블로그 병렬 검색
- Gemini JSON mode로 검색 결과 → 구조화된 행사 정보 추출
- confidence(high/medium/low) 기반 카드 필터링
- 서울 지역 필터 + 링크 출처 표시 구분 (공식 사이트 vs 뉴스/블로그)

---

## 2. 피벗 배경: 서울 열린데이터광장 → 네이버 검색

### 서울 열린데이터광장 API의 한계

```
- 데이터 커버리지: 공식 등록 행사만 포함, 소규모/팝업/단기 행사 누락
- 최신성: 당일~이번 주 행사가 미등록된 경우 많음
- 조회 구조: 전체 목록 fetch 후 클라이언트 사이드 필터링
  → fetch_size = limit × 6이어도 조건 맞는 행사가 없는 경우 발생
```

### 네이버 뉴스+블로그 방식의 장점
- 행사 공식 발표, 홍보 포스팅, 후기까지 실시간 반영
- 키워드+날짜 토큰 쿼리로 의도에 맞는 결과 수집 가능
- 공식 API 없이도 광범위한 행사 커버리지 확보

---

## 3. 구현 내용

### 3-1. 플로우

```
사용자 메시지
  → parse_date_expression(): 날짜 표현 → (date_from, date_to, date_token)
      예: "이번 주말" → ("2026-03-28", "2026-03-29", "2026년 3월 28일 29일")
  → 카테고리/지역/무료 패턴 매칭 (정규식)
  → build_event_queries(): 3개 검색 쿼리 생성
      예: ["서울 전시 무료 2026년 3월 28일 29일",
           "서울 무료 전시 2026년 3월 28일 29일",
           "전시 무료 2026년 3월 28일 29일"]
  → 병렬 검색 (asyncio.gather, 최대 6 요청)
      ├── search_news(query) × 3
      └── search_blog(query, suffix="") × 3
  → LLM 구조화 추출 (Gemini 2.5 Flash, JSON mode)
      → [{ title, category, place_name, date_start, date_end,
           price, detail_url, summary, confidence }, ...]
  → confidence 필터: high/medium만 카드 표시 (최대 5개)
  → 날짜 미추출 항목에 파싱된 날짜 보완
  → events 카드 블록 + text_stream 요약
```

### 3-2. 날짜 파싱 (`date_parser.py`)

| 입력 | date_from | date_to | query_token |
|------|-----------|---------|-------------|
| 이번 주말 | 2026-03-28 | 2026-03-29 | 2026년 3월 28일 29일 |
| 다음 주 | 2026-03-30 | 2026-04-05 | 2026년 3월 5주 |
| 3월 말 | 2026-03-21 | 2026-03-31 | 2026년 3월 말 |
| 봄 | 2026-03-01 | 2026-05-31 | 2026년 봄 |
| 오늘 | 2026-03-25 | 2026-03-25 | 2026년 3월 25일 |

### 3-3. 검색 쿼리 생성 (`build_event_queries`)

```python
queries = [
    f"{loc} {cat}{free} {date_token}{kw}",  # 지역 + 카테고리 + 날짜
    f"서울{free} {cat} {date_token}{kw}",   # 서울 고정
    f"{cat}{free}{kw} {date_token}",         # 카테고리 + 날짜만
]
```

### 3-4. LLM 구조화 추출

- 모델: Gemini 2.5 Flash (`response_mime_type="application/json"`)
- 입력: 뉴스+블로그 결과 텍스트 (최대 4000자)
- 출력: 행사 JSON 배열 + confidence 레벨
- 서울 외 지역 행사 제외 지시 포함

## 4. 트러블슈팅

| 문제 | 원인 | 해결 |
|------|------|------|
| 서울 열린데이터광장 이번 주 행사 조회 안 됨 | 당일/단기 행사 미등록, fetch 한도 초과 | 네이버 검색 방식으로 피벗 |
| 예매 링크 클릭 시 관련 없는 뉴스 기사 이동 | `detail_url`이 공식 사이트가 아닌 뉴스 기사 URL | URL 도메인 판별 → 버튼 레이블 분기 (공식 사이트 vs 관련 기사/블로그) |
| 서울 외 지역 행사 포함 | LLM 추출 시 지역 필터 없음 | EXTRACT_SYSTEM 프롬프트에 서울 한정 명시 |

## 5. 개선 필요 사항

### 5-1. 구조적 한계

**공식 사이트 URL 미확보**
- 네이버 뉴스/블로그 링크가 `detail_url`로 설정됨
- 예매·상세 정보 접근을 위한 공식 URL은 LLM이 본문에서 추출해야 하는데 항상 포함되지 않음
- 근본 해결: KOPIS, 인터파크, 예스24 등 공연 예매 사이트 직접 연동

**LLM 추출 정확도 한계**
- 검색 결과 텍스트 품질에 따라 날짜·장소 추출 실패 가능
- confidence=low 항목이 많을 경우 결과가 0건으로 표시됨
- 근본 해결: 공식 행사 API 연동 (KOPIS, TourAPI, 서울문화포털) 병행

### 5-2. 즉시 개선 가능
- 복수 날짜 표현 지원: "3월 28일부터 30일까지" 등 범위 입력
- 카카오/인터파크 행사 검색 API 추가 (키 발급 시)
- confidence=low 항목도 텍스트로만 안내 (현재 드롭)

---

## 6. 결론

네이버 뉴스+블로그 기반 이벤트 검색은 공공 API 대비 **실시간성과 커버리지**에서 유의미한 개선을 보였다. 날짜 파싱과 쿼리 생성 파이프라인은 다양한 표현을 잘 처리하고, LLM 구조화 추출도 기본적으로 동작한다. 다만 **예매 링크 품질**과 **추출 안정성**은 공식 공연 API(KOPIS 등) 연동으로 보완이 필요하다.
